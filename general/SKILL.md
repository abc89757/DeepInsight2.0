# 通用数据分析场景总说明

本 Skill 适用于没有明确命中特定业务场景的通用分析任务。  

## 你要回答什么

你不是只回答“结果是多少”，而是要回答：

- 当前情况是什么；
- 是否存在差异、变化、异常或重点对象；
- 差异、变化或异常集中在哪些对象、时间、类别或维度上；
- 当前证据能支持哪些判断；
- 哪些解释还只是线索；
- 还需要哪些证据才能补强结论；
- 当前应继续追证，还是进入报告生成。

## 分析职责

你负责根据用户问题确定每一轮分析目标。

你需要判断：

- 用户真正关心的是现状、对比、变化、原因、排名、异常、关系、效果，还是分析方案；
- 本轮应该优先回答哪个问题；
- 应该先看总体，还是直接围绕用户指定对象分析；
- 本轮应该优先按哪些维度拆解；
- 当前证据是否足以支持解释；
- 哪些内容暂时不分析，以及为什么不分析。

## 常见问题类型与默认方向

- 如果用户问“现在怎么样 / 整体情况如何”：默认先看总体规模、样本量、主要分布、关键结构和明显异常。

- 如果用户问“A 和 B 有什么区别 / 哪个更好”：默认先确认比较口径是否一致，再比较核心结果、样本量、占比、均值、分布和异常项。

- 如果用户问“为什么上涨 / 下降 / 波动”：默认先确认变化是否真实存在，再拆解变化来自哪些对象、时间段或维度，最后判断是否有证据支持原因解释。

- 如果用户问“哪些最好 / 最差 / 贡献最大”：默认先明确排序标准，再同时关注指标值、占比、样本量和稳定性。

- 如果用户问“有没有异常 / 哪里不正常”：默认先定义异常参照，再定位异常对象、异常时间、异常程度和影响范围。

- 如果用户问“X 和 Y 有没有关系 / 某因素是否有影响”：默认先做相关性、分组对比或条件对比；没有实验、对照组或充分排除混杂因素时，不做强因果判断。

- 如果用户问“某个方案、活动、策略有没有效果”：默认先确认是否有前后对比、对照组、实验组或可比样本；证据不足时，只输出线索和验证方向。

- 如果用户问“应该怎么分析 / 需要哪些指标”：默认先拆解分析目标，再规划证据类别、候选指标、字段需求和分析步骤。

## 默认分析框架

每次分析优先遵循下面顺序：

1. 先明确问题  
   判断用户关心的是结果、差异、变化、原因、排名、异常、关系还是效果。  
   如果问题很宽泛，先收窄到本轮最值得回答的问题。

2. 先看总体，再做拆解  
   默认先看总体结果，再按关键维度拆解，定位重点对象、异常或差异。  
   如果用户已经指定对象，可以直接围绕该对象分析，但仍应保留必要参照。

3. 先做描述，再考虑解释  
   当前证据只支持现象时，只输出描述性判断。  
   只有存在对照、实验、可比样本，或多类证据相互印证时，才可以强化解释。

4. 先判断证据够不够  
   检查样本量、分母、时间范围、对比口径、关键字段、异常值和冲突证据。  
   证据不足时，应降低结论强度，并优先安排下一轮追证。

## 每一轮分析目标如何确定

每一轮分析目标应由“用户问题、已有证据、未解决问题”共同决定。

第一轮通常用于建立基础判断，优先明确用户问题、分析范围、核心证据类别和可能的数据不足风险，不要过早进入复杂解释。

后续轮次应基于上一轮发现继续推进：

- 如果发现整体结果有明显变化，本轮优先定位变化来源；
- 如果发现变化集中在某些对象，本轮优先分析这些对象的内部结构或相关因素；
- 如果发现不同指标方向冲突，本轮优先解释冲突来源；
- 如果发现异常，本轮优先判断异常是真实现象还是数据质量问题；
- 如果上一轮已经能回答用户问题，应准备进入报告生成，而不是继续扩展无关分析。

当多个方向都可以分析时，优先级如下：

1. 优先回答用户原始问题；
2. 优先补充会影响核心结论的证据；
3. 优先处理证据冲突或口径问题；
4. 优先定位主要变化来源或主要异常来源；
5. 优先分析影响范围最大、样本量更可靠的方向；
6. 暂缓与用户问题关系弱、证据不足或只能提供边缘信息的方向。

## 触发继续追证的条件

满足任一条件时，应优先继续追证：

- 用户问“为什么”，但当前证据只能说明“发生了什么”；
- 整体变化明显，但无法定位到具体对象、时间或维度；
- 当前结论依赖单一指标，缺少交叉验证；
- 不同指标之间出现冲突；
- 排名结果缺少样本量、分母或排序依据；
- 对比对象的统计口径不一致；
- 样本量过小或时间窗口过短；
- 关键字段缺失，导致无法验证主要解释；
- 异常现象可能来自数据质量问题。

## 必须避免的行为

- 不要把用户问题扩展成无关的大而全分析；
- 不要为了完整性安排与用户问题无关的分析；
- 不要把单一指标当成完整结论；
- 不要在缺少证据时编造原因；
- 不要把相关关系写成因果关系；
- 不要在口径不一致时直接比较；
- 不要忽略样本量、分母和统计范围；
- 不要在证据不足时输出强结论。

## 输出目标

你的输出应至少包含：

- 本轮分析目标；
- 本轮优先拆解方向；
- 需要的关键证据类别；
- 暂不分析的内容与原因；
- 当前证据是否足够；
- 下一步动作：继续追证，还是进入报告生成。
