# 商品销售分析图表规则

## 文档定位

本文档用于说明在商品销售分析场景中，你应该根据用户问题、分析目标和已获得的数据结果，选择什么图表、展示什么信息、如何避免误导性表达，使用图表帮助用户更快理解：

- 销售结果变化
- 商品结构差异
- 类目 / 渠道 / 门店 / 地区对比
- 价格、折扣、退款变化
- 库存风险
- 客户价值分层
- 促销和广告效果
- 异常点和重点对象

## 总体原则

- 图表必须服务于用户问题。
- 不需要图表的问题不要强行画图。
- 每张图只能表达一个主要信息。
- 图表标题必须说明“分析对象 + 指标 + 时间范围 / 维度”。
- 横轴、纵轴、单位必须清楚。
- 如果数据经过过滤、聚合、抽样或 Top N 截断，必须在图表说明中写出。
- 如果样本量不足、字段缺失或口径不完整，图表仍可展示，但必须降低解释强度。
- 不要用图表制造没有证据的趋势、因果或异常判断。
- 不要在一张图里塞太多维度，复杂问题应拆成多张图。

## 图表优先级

默认优先选择简单、可解释的图表：

1. 折线图：看时间趋势。
2. 柱状图：看类别对比和排名。
4. 饼图 / 环形图：看少量类别的占比。

---

# 一、折线图

## 适用信息

折线图适合表示随时间变化的趋势。

适合回答：

- 销售额最近是上升还是下降？
- 订单量是否持续变化？
- 客单价是否在某段时间波动？
- 促销前后销售趋势有没有变化？
- 某类目或渠道的销售趋势是否异常？
- 退款率、折扣率是否逐渐升高？

## 适用指标

- 销售净额（net_sales）
- 订单数（orders）
- 售出件数（units_sold）
- 净额口径客单价（aov_net）
- 净额口径平均售价（asp_net）
- 折扣率（discount_rate）
- 金额口径退款率（refund_rate_amt）
- 毛利率（gross_margin）
- 广告花费（ad_spend）
- 广告支出回报率（roas）
- 库存可支撑天数（days_on_hand）

## 适用数据结构

需要至少包含：

- 时间字段
- 一个数值指标

示例数据结构：

| 日期       | 销售净额 | 订单数 |
| ---------- | -------: | -----: |
| 2026-05-01 |    12000 |     86 |
| 2026-05-02 |    13500 |     91 |

## 推荐使用方式

- 单指标趋势：一条折线。
- 多对象趋势：多条折线，例如不同渠道销售额趋势。
- 当前周期 vs 对比周期：两条折线对比。
- 促销前后变化：在图中标注促销开始时间。

## 不适用情况

- 类别过多，例如几十个商品同时画折线。
- 时间点过少，例如只有 1~2 个日期。
- 时间间隔不均匀但没有说明。
- 数据缺失严重，容易造成虚假趋势。

## 注意事项

- 时间粒度要统一，例如按天、按周或按月。
- 当前周期未结束时，不要直接和完整历史周期比较。
- 多条折线超过 5 条时，应只展示 Top N 或重点对象。
- 如果趋势来自数据回填延迟，必须提示。

# 二、柱状图

## 适用信息

柱状图适合表示不同类别之间的数值对比。

适合回答：

- 哪些商品卖得最多？
- 哪些类目销售额最高？
- 哪些渠道贡献最大？
- 哪些门店表现最好或最差？
- 哪些地区销售额最高？
- 哪些商品退款率最高？
- 哪些 SKU 毛利贡献最大？

## 适用指标

- 销售净额（net_sales）
- 销售总额（gross_sales）
- 订单数（orders）
- 售出件数（units_sold）
- 毛利润（gross_profit）
- 毛利率（gross_margin）
- 退款金额（refund_amount）
- 金额口径退款率（refund_rate_amt）
- 折扣金额（discount_amount）
- 折扣率（discount_rate）
- 促销销售额（promo_sales）
- 广告花费（ad_spend）
- 广告转化数（conversions）

## 适用数据结构

需要至少包含：

- 一个类别字段
- 一个数值指标

示例数据结构：

| 类目 | 销售净额 |
| ---- | -------: |
| 手机 |   120000 |
| 服饰 |    86000 |

## 推荐使用方式

- 排名对比：按指标降序排列。
- Top N 展示：只展示前 5、前 10 或前 20。
- 正负变化对比：展示增长和下降的类别。
- 横向柱状图：类别名称较长时优先使用。

## 不适用情况

- 类别数量过多且没有筛选。
- 指标之间量纲完全不同，却放在同一柱状图中。
- 类别本身没有业务意义，例如大量 unknown、not set。

## 注意事项

- 必须排序，除非类别有固定顺序。
- 类别超过 10 个时，默认展示 Top N。
- 如果“其他 / 未知”占比很高，应单独提示数据质量问题。
- 对比销售额时，可以同时在报告中补充占比，避免只看绝对值。

---

# 三、分组柱状图

## 适用信息

分组柱状图适合比较多个类别在不同分组下的表现。

适合回答：

- 不同渠道中，各类目的销售表现有什么差异？
- 不同门店中，哪些商品表现最好？
- 当前周期和对比周期，各类目变化如何？
- 促销前后，不同商品销售是否变化？

## 适用指标

- 销售净额（net_sales）
- 订单数（orders）
- 售出件数（units_sold）
- 毛利率（gross_margin）
- 折扣率（discount_rate）
- 退款率（refund_rate_amt）

## 适用数据结构

需要至少包含：

- 一个主类别字段
- 一个分组字段
- 一个数值指标

示例：

| 类目 | 周期     | 销售净额 |
| ---- | -------- | -------: |
| 手机 | 当前周期 |   120000 |
| 手机 | 对比周期 |   100000 |

## 不适用情况

- 主类别过多。
- 分组过多。
- 多个指标同时比较，导致图表拥挤。

## 注意事项

- 主类别建议不超过 8 个。
- 分组建议不超过 3 个。
- 如果类别太多，应改为多张柱状图或只展示重点类别。

---

# 四、饼图 / 环形图

## 适用信息

饼图和环形图适合表示少量类别的整体占比。

适合回答：

- 各类目销售占比是多少？
- 各渠道订单占比是多少？
- 促销销售占整体销售多少？
- 新客和老客占比是多少？
- 某几个核心品牌贡献了多少销售额？

## 适用指标

- 单品销售贡献占比（sku_sales_share）
- 类目销售贡献占比（category_sales_share）
- 品牌销售贡献占比（brand_sales_share）
- 渠道订单占比（channel_order_share）
- 促销销售占比（promo_sales_share）
- 新客户数（new_customers）
- 复购客户数（returning_customers）

## 适用数据结构

需要至少包含：

- 一个类别字段
- 一个占比或可换算为占比的数值字段

示例：

| 渠道     | 订单数 |
| -------- | -----: |
| 线上商城 |   1200 |
| 门店     |    800 |

## 不适用情况

- 类别超过 5 个。
- 各类别占比接近，难以比较。
- 需要比较多个时间点的占比变化。
- 需要精确展示排名。

## 注意事项

- 类别超过 5 个时，优先改用柱状图。
- 如果有“其他”类别且占比很高，应提示该图解释力有限。
- 饼图只能表达占比，不适合表达趋势。

---

# 五、不同用户问题的图表选择

## 销售额为什么下降？

推荐图表：

1. 折线图：销售净额、订单数、客单价趋势。
2. 柱状图：各类目 / 商品 / 渠道的销售变化贡献。
3. 分组柱状图：当前周期 vs 对比周期的核心指标。
4. 散点图：折扣率与销售额、退款率与销售额的关系。

不推荐：

- 饼图。因为销售下降是变化问题，不是单纯占比问题。

## 哪些商品卖得最好？

推荐图表：

1. 横向柱状图：Top N 商品销售净额。
2. 柱状图：Top N 商品销量。
3. 气泡图：销量、毛利率、销售额综合表现。
4. 表格：Top N 商品详细指标。

不推荐：

- 饼图展示太多 SKU。
- 折线图展示大量商品趋势。

## 哪些类目贡献最大？

推荐图表：

1. 柱状图：类目销售净额排名。
2. 堆叠柱状图：类目结构随时间变化。
3. 饼图：类目数量较少时展示占比。
4. 热力图：地区 × 类目或渠道 × 类目销售强度。

## 促销有没有效果？

推荐图表：

1. 折线图：促销前后销售趋势。
2. 分组柱状图：促销期 vs 非促销期核心指标。
3. 双轴图：促销销售额与毛利率 / 折扣率。
4. 柱状图：促销商品销售额和毛利率对比。

注意：

- 促销效果不能只看销售额，还要看毛利率、退款率和促销销售占比。
- 没有对照组时，只能说“促销期间表现”，不要说“促销导致增长”。

## 库存是否有问题？

推荐图表：

1. 柱状图：库存可支撑天数最高 / 最低的商品。
2. 散点图：库存可支撑天数 × 平均日销量。
3. 热力图：商品 × 日期的缺货分布。
4. 折线图：库存件数与销量趋势。

注意：

- 销量下降但库存低，不能直接判断需求下降。
- 库存高但销量弱，应关注压货风险。
- 库存低且销量高，应关注缺货风险。

## 客户复购怎么样？

推荐图表：

1. 柱状图：新客与老客销售贡献。
2. 折线图：复购率趋势。
3. cohort 留存热力图：不同首购批次后续留存表现。
4. RFM 分层柱状图：不同客户层级人数或销售贡献。

注意：

- 没有客户 ID 时不能做客户复购分析。
- cohort 样本量过小时应降低结论强度。

## 广告投放是否有效？

推荐图表：

1. 折线图：广告花费、转化数、ROAS 趋势。
2. 散点图：广告花费 × ROAS。
3. 柱状图：不同广告活动的 CPA 或 ROAS 对比。
4. 双轴图：广告花费与销售额 / ROAS。

注意：

- 没有广告归因销售额时，不计算 ROAS。
- ROAS 高不一定代表利润高，还要结合毛利率。
- 归因窗口不明确时，应提示口径限制。

# 六、图表输出要求

每张图表应附带以下信息：

- 图表标题
- 图表类型
- 使用指标
- 使用维度
- 时间范围
- 数据口径
- 是否经过过滤或 Top N 截断
- 图表结论
- 结论强度
- 数据限制

## 图表说明写法

每张图表下方应写 1~3 句话解释。

推荐写法：

- “图中可以看到，……”
- “这说明……”
- “但由于……，该结论强度为中 / 弱。”

禁止写法：

- “显然证明……”
- “一定是因为……”
- “完全说明……”
- “毫无疑问……”

---

# 八、避免误导的规则

- 不要把不同量纲的指标放在同一坐标轴。
- 不要用饼图展示太多类别。
- 不要用折线图连接没有时间顺序的类别。
- 不要在样本量很小的情况下画复杂图。
- 不要用颜色暗示好坏，除非业务规则明确。
- 不要把相关关系解释为因果关系。
- 不要隐藏 unknown、其他、缺失这类类别；如果占比高，应提示数据质量问题。
- 不要只展示支持结论的数据，而忽略相反证据。
- 不要在没有对比周期时画“增长 / 下降”图。
