# 分组内最优对象识别计算指南

## 场景说明
本文件适用于在多维度数据中，根据一个指定的排序指标，筛选出每个分组内表现最佳的具体对象的计算场景。此计算模式的核心是“分组-排序-取极值”，适用于电商品类销量排名、零售品牌明星产品、服务网点评分最优门店、内容平台热门文章等各类需要按组别找出冠军对象的业务分析。

## 核心概念定义
- **分组维度（Group Dimension）**：用于划分数据集合的类别字段。例如：`品牌`、`产品品类`、`城市`、`服务网点`、`用户分群`。
- **对象维度（Object Dimension）**：在每个分组内，需要进行比较和筛选的具体实体字段。例如：`具体车型`、`SKU商品`、`门店名称`、`具体文章标题`。
- **排序指标（Sorting Metric）**：用于衡量对象在所属分组内表现优劣的量化指标。例如：`成交量`、`销售额`、`评分`、`点击量`、`客户满意度`。
- **最优对象筛选规则**：默认定义为在每个分组内，`排序指标`取值最高的对象。若需筛选表现最差的对象，应明确说明“筛选每组内指标值最低的对象”。

## 统一计算口径
1.  **目标明确**：计算的目标是输出“每个分组维度值下，排序指标值最高的对象维度值及其指标值”。
2.  **数据粒度**：计算所需的数据必须至少包含三列：一个`分组维度`字段，一个`对象维度`字段，和一个可聚合的`排序指标`字段（或用于计算该指标的原始字段）。数据应为明细记录或可直接聚合的记录。
3.  **计算流程**：
    - **第一步：数据预处理**。对`分组维度`字段进行标准化清洗（见下文规则），确保同一分组的标识一致。
    - **第二步：聚合计算**。按`(分组维度, 对象维度)`分组，对`排序指标`进行聚合（例如：`SUM(成交量)`、`AVG(评分)`、`MAX(最后一次更新时间)`）。
    - **第三步：组内排序**。在每个`分组维度`内部，按聚合后的`排序指标`进行降序排序。
    - **第四步：筛选最优**。取每个`分组维度`内排序后的第一条记录，即为该组的最优对象。
4.  **口径一致性**：在整个计算过程中，`分组维度`、`对象维度`和`排序指标`的定义必须保持全文档一致。若分析过程中需要切换排序指标（如从按成交量排改为按成交额排），应视为新的计算任务。

## 数据质量规则与降级策略

### 字段缺失与空值处理
- **关键字段缺失**：如果记录缺少`分组维度`或`对象维度`字段，该条记录**不应参与**任何分组或聚合计算。
- **排序指标缺失或为零**：
    - 如果某条记录的`排序指标`字段为空或为0，在聚合时，其值将如实计入（例如，销售额为0的订单将拉低该分组的平均销售额）。
    - 如果某个`(分组, 对象)`组合聚合后的`排序指标`为空或0，该组合仍可参与组内排序，但排名将靠后。
    - **输出策略**：当某个`分组维度`下所有对象的排序指标聚合结果均为空或0时，该分组应在最终结果中以“无有效数据”或类似标记呈现，或直接省略，并在结果说明中注明。

### 分组字段标准化规则
- **处理原则**：在计算前，必须对`分组维度`字段值进行标准化，以合并同一实体的不同表述。
- **标准化操作示例**：
    - **大小写统一**：将所有值转换为大写或小写（例如，`NISSAN`、`Nissan` -> `NISSAN`）。
    - **去除多余字符**：去除值末尾的“空格”、“.”、“-”等非核心字符。
    - **合并常见变体**：建立明确的合并映射规则。例如，可规定将“`LAND ROVER`”与“`LANDROVER`”合并；将“`MERCEDES-BENZ`”、“`MERCEDES`”统一为“`MERCEDES-BENZ`”。**具体映射表应根据业务数据字典制定，而非臆造。**
- **降级策略**：
    - **保守策略（默认）**：若无法确定某些变体是否属于同一分组（如`FORD`与`FORD TRUCK`，可能属于不同业务线），则分别计算，并在结果中同时列出，供决策者判断。
    - **激进策略**：仅在业务规则明确支持且有标准化映射表时，将所有已知变体强制合并计算。此策略需在结果中明确说明合并口径。

### 估算口径说明
- 如果用于判断“最优”的`排序指标`是估算值（例如，使用“最近7天销量”估算“月度畅销度”，或用“样本评分”估算“整体满意度”），则必须在结果的显著位置标记该指标为“估算口径”，并简要说明估算依据。

## 示例（抽象化）
假设有一个销售数据集，需要“找出每个产品品类下销售额最高的具体商品”。

1.  **分组维度**：`产品品类`（如“电子产品”、“家居用品”）
2.  **对象维度**：`商品ID`或`商品名称`
3.  **排序指标**：`销售额`（聚合计算：`SUM(成交金额)`）
4.  **标准化规则**：对`产品品类`字段进行去空格、统一大小写处理。
5.  **计算流程**：
    - 预处理：将“电子产品”、“电子产品 ”、“electronics”统一为“电子产品”。
    - 聚合：计算每个`(产品品类, 商品)`的总销售额。
    - 排序与筛选：在“电子产品”品类内，按总销售额降序，取排名第一的商品及其销售额。
6.  **输出结果**：一个表格，列出每个标准化后的产品品类、其销售额最高的商品名称/ID，以及对应的销售额。

## 使用提醒
- **禁止直接套用SQL**：本指南描述计算逻辑，具体实现需根据数据库表结构和SQL语法调整。
- **禁止臆造字段**：所有计算必须基于数据源中实际存在的字段。
- **结果解读**：此计算结果仅反映在特定排序指标下的分组内冠军。要全面评估对象表现，需结合其他指标（如增长率、利润率、用户评价等）综合判断。
- **数据时效性**：计算结果反映的是所用数据时间窗口内的状态，需注意数据的时间范围，并在结论中说明。
