# 分组内最优对象识别报告模板

本模板专为 **“在明确分组维度下，筛选各组内表现最优的具体对象”** 这一分析范式设计。它提供了一套标准化的报告结构、表达规范与注意事项，以指导分析师高效、清晰地呈现此类分析的结果。

---

## 报告标题

标题应直接点明“分组维度”、“识别目标”及可选的“核心排序指标”。

**公式**：`各【分组维度】的最优【对象类型】识别分析` 或 `各【分组维度】最【核心排序指标】的【对象类型】识别分析`。

**示例**：
- 各品牌的最优车型识别分析
- 各区域销售额最高产品识别分析
- 各品类评分最佳服务商识别分析

**要求**：
- 必须包含“分组维度”和“对象类型”。
- 可包含“最优”、“最高”、“最佳”等识别含义词。
- 可包含“核心排序指标”（如“销售额最高”），但非必须。
- **禁止**：使用泛泛的“数据分析报告”、“销售分析”等标题。

---

## 执行摘要

篇幅建议：100~150字。

**必须包含**：
1.  **识别目标**：一句话说明本次分析要解决的问题。
2.  **核心结果**：直接陈述识别结果的整体模式或关键发现（例如：“在【分组维度】下，以【核心指标】衡量，各组最优对象呈现【模式描述，如头部集中、特征鲜明】”）。
3.  **全局冠军**：指出整体表现最突出的冠军（全局Top 1）及其关键数据。
4.  **关键限制**：一句核心的数据质量或口径限制说明。

**推荐写法**：
```md
本次分析旨在识别【用户问题概述】。在【分组维度】范围内，以【核心指标】衡量，各组最优对象呈现【模式描述】（结论强度：强/中/弱）。整体表现最优的冠军是【全局Top 1简述】。需要注意的是，【核心限制简述】，因此【受影响部分】结果需谨慎解读。
```

---

## 问题与范围

建议篇幅：60~100字。

**必须写清**：
- 用户原始问题。
- 本次实际识别目标（即“找出每个【分组维度】中，【核心指标】最高的【对象类型】”）。
- 数据的时间范围（如有）。
- **核心排序指标的具体口径**（例如：“成交量”、“总成交金额”、“用户评分均值”）。
- **参考指标及其口径**（例如：“总成交金额”作为“成交量”的参考）。
- **数据覆盖范围**（例如：覆盖了【X】个分组）。

**写法要求**：
- 聚焦于定义本次识别任务的边界。
- **降级说明**：若数据不完整（如仅能覆盖部分分组），必须在此说明“本报告仅识别并展示了【X】个分组中的【Y】个分组的最优对象”。
- 不要复述分析过程。

---

## 核心发现

这是报告的**绝对主体**，篇幅应占全文50%以上。必须围绕“识别结果”展开，并包含以下三个子模块。

### 子模块一：排名总览
- **目标**：直接回答用户问题，呈现识别结果。
- **内容**：**必须包含一个汇总表格**。表格列应至少包括：
    - `分组维度`
    - `最优对象（按核心指标）`
    - `核心指标值`
    - *可选：`参考指标值`（如：总成交额、平均分）*
- **降级策略**：
    - **分组过多**（如 > 15个）：正文表格仅展示 **Top N 和 Bottom N（或有业务代表性的分组）**，并注明：“完整各分组排名详见【附件/数据文件路径】”。
    - **数据不完整**：在表格或其说明中明确标注哪些分组因数据问题结果可能不准确或缺失。
- **写作要求**：
    - 表格后必须紧跟2-3句解读，说明表格揭示的**主要模式**（如“从核心指标看，冠军高度集中于某几类对象”）。
    - 禁止：仅罗列表格数据而不做解读。

### 子模块二：模式解读
- **目标**：解读排名结果所反映的整体规律和特点。
- **内容**：选择以下一点或几点进行阐述：
    1.  **头部格局**：哪些类型/属性的对象（如某类商品、某种价位）在多数分组中占据领先。
    2.  **指标差异**：如果分析了不止一个排序指标（如同时看销量和销售额），解读其排名差异所反映的业务特点（例如：“销量冠军与销售额冠军不一致，表明单价差异显著”）。
    3.  **分布特点**：最优对象是高度集中于少数类型，还是广泛分散。
- **写作要求**：
    - 解读必须基于排名总览中的数据表格，避免空泛描述。
    - 重点在于解释模式的含义，而非深入分析原因。
    - **禁止**：对不同分组的冠军进行无意义的跨组绝对值比较（例如，不比较A分组冠军的销量和B分组冠军的销量高低）。

### 子模块三：标准化问题影响（关键风险提示）
- **目标**：专门、突出地强调“分组维度”数据质量对结果可靠性的核心影响。
- **内容**：
    1. 指出数据中分组字段（如品牌名、品类编码）存在的不一致写法（如缩写、全称、带后缀）。
    2. 解释这如何导致同一实体被拆分为多个分组，从而影响“为每个分组都找到最优对象”这一结论的完整性。
- **写作要求**：
    - **这是此类场景报告的必选模块**。
    - 必须具体描述问题，评估其对结果的影响程度。
    - 通常应与“限制与数据问题”章节呼应。

### 图表与表格引用原则
1.  **核心排名表格**：**必须使用 Markdown 表格**呈现识别结果，如“子模块一”所示。
2.  **辅助可视化**：如需可视化，可使用条形图对比各分组排序指标值，或使用饼图展示最优对象类型的构成分布。
    - 引用时使用：`![图表标题](图片地址)`。
    - 图表应放在与其结论最相关的位置，并附简要说明。
    - **禁止**：编造图表内容或引用不存在的地址。

---

## 关键证据

关键证据是对全文核心信息的压缩汇总，用于快速浏览。对于本场景，**必须包含一个“分组最优对象总览表”**。

**推荐表格结构**：
| 分组维度 | 最优对象（按核心指标） | 核心指标值 | 参考指标 (如：价值) | 备注 (如：标准化问题) |
| :--- | :--- | ---: | ---: | :--- |
| | | | | |

**要求**：
- 此表是核心发现中详细排名表格的精炼版。
- 表格下方应简要说明核心发现的主要模式。
- 可包含其他关键的整体性指标（如：识别出的分组总数、冠军们的指标总和/均值等），但不应替代核心发现中的详细表格和解读。

---

## 业务解释

这一段回答“这些排名数字可能意味着什么”。

**建议写法**：
- 解读排名模式**可能的业务含义**（例如：“冠军商品集中在【某类】，可能说明这类商品具有更强的市场接受度或流通性。”）。
- 指出对决策的潜在支持方向（例如：“此排名可为品类引进或资源投放提供客观参考。”）。
- 提示需结合其他信息（如利润、库存、评价）做进一步判断。

**写作要求**：
- **禁止**：将描述性排名直接推断为确定性因果（例如，不能写“A产品销量最高，因此它是最好的产品”）。
- 允许使用“可能说明”、“暗示”、“可作为...的参考”等谨慎表述。

---

## 限制与数据问题

**必须单独成节**，且置于“结论与建议”之前。

**首要且必须包含的内容**：
1.  **分组维度标准化问题**：重申这是影响结果完整性的**首要风险**，并简要说明已发现的字段不一致情况。
2.  **排序指标口径限制**：说明“最优”的衡量指标的具体定义是否清晰、一致。

**其他可能内容**（根据实际情况添加）：
- 数据时间范围不明确。
- 对象分类标准（如是否包含特定类型）。
- 分组内数据是否完整（例如，是否只返回了 Top 1 对象，而非全量明细）。
- 核心发现中仅展示了部分分组，完整数据已另存。

**写法要求**：
- 具体，不写空话。
- 写清限制主要影响“子模块一：排名总览”结果的精确性与完整性。

---

## 结论与建议

建议拆成两部分。

### 结论摘要
- 用2-3句话总结最核心的识别结果和模式。
- 回扣用户原始问题，明确回答“各分组的最优对象是谁”。
- 如果存在数据质量问题，需提示其对结论可信度的总体影响。

### 后续建议
建议按优先级写：
1.  **数据治理建议**：如存在分组维度不一致问题，应首先建议进行数据清洗与合并，以获得更精确的排名结果。
2.  **分析深化建议**：如用户对特定分组或差异感兴趣，可建议进一步分析（例如：“可对头部【N】个冠军进行共性特征分析”）。
3.  **决策验证建议**：说明当前结论可支持何种初步决策，并提示需结合哪些其他信息进行验证。

---

## 默认篇幅建议

侧重于“核心发现”：
- 执行摘要：100~150字
- 问题与范围：60~100字
- **核心发现：300~500字**（含1个主表格，展示5-15个关键分组行）
- 关键证据：1个总览表，4-8条关键信息
- 业务解释：100~200字
- 限制与数据问题：80~150字
- 结论与建议：100~200字

如果用户问题非常聚焦（如“各品牌销量冠军是谁”），可压缩其他部分，将篇幅集中在核心发现的表格和解读上。

---

## 报告生成时的硬约束

- 不得新增上游未提供的事实。
- 不得把弱证据写成强结论。
- 不得省略限制说明。
- 同一指标全篇保持同一口径。
- **若结论依赖估算或衍生指标，必须在使用处提示“估算值”**。
- **若上游数据中存在冲突证据（如基于不同指标的矛盾排名），必须在报告中说明此差异，而非只选择其一**。
- 若上游只提供单期数据，不得写趋势判断（如“增长”、“变化”）。
- **必须突出强调“分组维度数据标准化”问题及其对结果完整性的影响**。
- **禁止**：对不同分组的“最优对象”进行无意义的跨组绝对值比较。
- **禁止**：将描述性排名推断为确定性因果。
- **如果当前数据不足以支撑“为每个分组都找到最优对象”的结论，必须在“问题与范围”和“限制”章节直接说明，而不是转移分析重点**。
