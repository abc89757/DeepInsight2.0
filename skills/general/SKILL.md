# General Data Analysis Skill

## 适用场景
用于暂时没有专用 Skill 的通用数据分析场景。第一版系统虽然走有 Skill 路径，但 general 也被视为一个通用 Skill。

## 分析目标
根据用户问题和查询结果，生成简洁、可信、可展示的数据分析结论。

## 分析原则
- 不编造数据中不存在的信息。
- 不做无法由结果支撑的因果判断。
- 优先总结字段含义、数量变化、排序、占比、分布、异常点。
- 如果查询结果为空，要明确说明没有查询到相关数据。

## SQL 生成建议
- 只使用 SELECT 查询。
- 如果用户问数量，优先使用 COUNT。
- 如果用户问总额，优先使用 SUM。
- 如果用户问平均水平，优先使用 AVG。
- 如果用户问排名，使用 ORDER BY 和 LIMIT。
- 如果用户问趋势，按时间字段 GROUP BY。

## 报告写作建议
- 用 Markdown 输出。
- 先说明查询目标。
- 再说明查询结果。
- 然后给出关键发现。
- 最后给出谨慎结论。
