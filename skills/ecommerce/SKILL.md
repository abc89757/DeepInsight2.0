# Ecommerce Analysis Skill

## 适用场景
用于电商、订单、销售、商品、用户购买行为等数据分析场景。

## 分析目标
根据用户问题，从订单、商品、用户、地区、时间等维度提取数据，生成结构化销售洞察报告。

## 常见指标
- 销售额：通常为实际支付金额或订单金额，可结合退款金额进行修正。
- 订单量：订单数量或支付成功订单数量。
- 客单价：销售额 / 订单量。
- 商品销量：商品售出数量。
- 品类销售额：按商品品类聚合的销售额。
- 地区销售额：按地区聚合的销售额。
- 退款率：退款金额 / 支付金额，或退款订单量 / 总订单量。

## 推荐分析维度
- 时间维度：按天、周、月分析趋势。
- 商品维度：分析商品销售排行、重点商品贡献。
- 品类维度：分析不同品类的销售结构。
- 地区维度：分析不同地区的销售差异。
- 用户维度：分析用户购买行为、复购情况。

## SQL 生成建议
- 优先使用 SELECT 查询，不允许修改数据库。
- 趋势类问题优先按时间字段 GROUP BY。
- 排名类问题使用 ORDER BY 和 LIMIT。
- 聚合分析时注意 COUNT、SUM、AVG 等函数。
- 如果存在订单状态字段，应优先筛选已支付或有效订单。
- 如果存在测试订单字段，应排除测试数据。

## 报告写作建议
- 先说明用户问题和查询目标。
- 再展示核心指标或查询结果概览。
- 然后总结主要发现。
- 如果发现异常，只能基于当前数据初步判断，不要编造原因。
- 最后给出结论和可选的进一步分析建议。
