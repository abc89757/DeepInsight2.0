# 取数与计算规则

## 基础取数

- 售价字段优先使用成交价；如果没有成交价，再使用挂牌价或价格字段。
- 颜色字段需要排除空值，必要时把大小写、空格或同义写法标准化。
- 如果有销售状态字段，优先分析已成交或已售车辆。
- 如果用户没有指定时间范围，先使用全部可用数据；如果数据量很大，可以在 SQL 中做聚合，避免把明细全部交给后续 Agent。

## 推荐 SQL 聚合

分析“不同颜色的汽车是否会影响售价”时，优先生成按颜色分组的聚合结果：

- count 作为样本量。
- avg(price) 作为平均售价。
- min(price)、max(price) 作为价格范围。
- 如果数据库支持 percentile 或中位数函数，可以计算 median(price)。

## 进一步控制

如果基础结果显示颜色差异明显，下一轮可以按品牌、车型、年份或里程分层，检查差异是否来自车辆结构不同。

## 缺失情况

如果缺少颜色或售价字段，需要返回可用字段，并说明无法直接回答颜色与售价关系。
