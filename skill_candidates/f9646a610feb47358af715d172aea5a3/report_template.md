# 分组维度下 Top/Bottom 识别与排名分析报告模板

本模板适用于“按特定维度（如品牌、类目、门店、渠道、客户群等）分组，识别组内在特定指标（如销量、销售额、活跃度等）下表现最好或最差的N个子项（如商品、车型、功能、用户等）”的分析场景。模板聚焦于报告的结构、表达规范与硬性约束，确保输出清晰、严谨且可复用。

## 报告标题
标题应直接回答“分组维度 + 排序目标 + 核心问题”。
示例：
- 各品牌销量最高的商品识别
- 各区域门店客流量 Top 5 及 Bottom 5 表现
- 各客户行业中使用频率最高的功能模块分析

要求：
- 明确包含核心分析动作（如“识别”、“排名”、“分析”）。
- 明确包含分组维度和排序指标。
- 避免使用“Top N 报告”等模糊表述。

---

## 执行摘要
篇幅：150-250字。
必须包含：
1.  本次分析所回应的具体业务问题。
2.  最关键的 2-3 条排名发现（例如：在A维度下，B项在C指标上位居第一；整体呈现X分布格局）。
3.  对排名结果可靠性的关键限制说明（例如：因数据一致性问题，部分分组结果可能不完整）。

写法：
- 结论先行，直接给出核心排名结果。
- 措辞客观，避免主观评价。
- 必须包含对结论强度的提示（强/中/弱），并简要说明依据或限制。

推荐写法：
```md
本次分析旨在回答【用户具体问题】。基于【关键指标】的排名结果如下：最主要的发现是，在【分组维度】下，【子项A】的【指标】表现最强，为【数值】（结论强度：【强/中/弱】）。整体上，【描述头部格局，如“头部主要由XX类型构成”】。需要注意的是，【关键限制，如“该排名受限于数据标准化质量”】，因此结论【强度说明】。
```

---

## 问题与范围
篇幅：80-150字。
必须写清：
- 用户原始问题。
- 本次实际回答的、经过范围收窄后的问题。
- 明确的分析时间范围（如果数据未提供，需注明）。
- 核心分组维度（如品牌）。
- 核心排序指标（如销量）。
- 辅助参考指标（如销售额、增长率等）。
- 关键统计口径（例如：销量指成交订单数还是出货件数？金额是否含税？）。

写法：
- 聚焦于“按什么维度、根据什么指标、找出什么”。
- 不展开分析背景或无关的业务上下文。

---

## 核心发现
本节是报告主体，必须直接、清晰地展示排名结果及其含义。

### 核心发现写作结构
建议包含 2-4 个小节，每个小节围绕一个主要的排名发现展开。

**每个小节必须包含：**
1.  **小节标题**：直接陈述该发现的核心结论（例如：“发现一：销量领先的头部商品以XX品类为主”）。
2.  **结论阐述**：用2-4句话说明该发现，必须引用关键排名数据。
3.  **局部排名数据表格**：清晰展示Top/Bottom N的排名情况。
4.  **表格解读**：解释表格数据说明了什么，以及对用户问题的含义。
5.  **结论强度与限制**：说明此结论的可靠性及受何种数据或分析限制影响。

**推荐小节结构：**
```md
### 发现一：【直接回答排名问题的结论，如“XX维度下指标最高的前N项是YY”】

（阐述发现，引用关键数字）

| 排名 | 分组维度 | 子项 | 核心排序指标 | 关键辅助指标1 | 关键辅助指标2 | 备注/说明 |
| ---: | :------- | :--- | ----------: | ----------: | ----------: | :-------- |
| 1    |          |      |             |             |             |           |
| ...  |          |      |             |             |             |           |

**解读**：
- 上表显示了在【分组维度】内，按【核心指标】排序的结果。
- 排名前N的子项主要集中在【某种类型或特征】，这说明了【业务现象，如该类商品流通性强】。
- 结论强度为【强/中/弱】，因为【说明理由，如样本量充足/存在未合并的重复分组项】。
```

## 图表引用与局部数据表格

### 图表引用
- 如果上游分析节点已生成图表（如分布图、占比图），可在相关结论处按需引用，使用Markdown图片语法：`![图表标题](图片地址)`。
- 每张图表下方需用1-2句话说明其展示的核心信息及其对结论的支持。
- 不得引用不存在的图片地址或编造图表。

### 局部排名数据表格要求
核心发现部分必须包含至少一个局部排名数据表格，用于直观展示Top/Bottom N的结果。

**表格字段建议：**
1.  **排名**：明确的序号。
2.  **分组维度**：当前分组所依据的维度（如“品牌”）。
3.  **子项**：被排名的具体对象（如“车型”）。
4.  **核心排序指标**：用于排名的指标（如“销量”），建议右对齐。
5.  **关键辅助指标1/2**：1-2个帮助理解排名业务意义的指标（如“销售额”、“市占率”），建议右对齐。
6.  **备注/说明**：对异常值、数据质量或业务背景的简要说明。

**控制规则：**
- 默认展示排名前5或前10的项目（Top N），或排名后5的项目（Bottom N）。
- 如果数据量很大或存在异常值，可以重点展示头部和尾部，并提示中间部分结果。
- 如果完整排名数据已保存至文件，应在表格下方注明：“完整排名详见结果文件【文件名】。”
- 表格应服务于当前小节的结论，避免堆砌无关指标。

## 关键证据
对全文核心排名发现和数据质量的浓缩汇总，建议使用Markdown表格。

| 证据主题           | 关键指标                                                         | 核心值/描述                                     | 数据/样本范围说明 | 强度   |
| ------------------ | ---------------------------------------------------------------- | ----------------------------------------------- | ----------------- | ------ |
| 排名结果有效性     | 样本覆盖的分组维度数量                                           | 共涵盖X个分组                                   |                   | 强/中/弱 |
| 数据标准化质量     | 存在变体或不一致写法的分组维度数量                               | 识别出Y处可能的不一致写法                       |                   | 强/中/弱 |
| 排名分布格局       | Top 3 子项占总体核心指标的贡献占比                               | XX%                                             |                   | 强/中/弱 |
| 分析范围限定       | 数据时间范围                                                     | 【明确时间范围或注明未限定】                    |                   | 强/中/弱 |

要求：
- 证据条数不少于3条。
- 每条证据应与核心发现中的某个结论或限制直接对应。
- 必须说明证据的强度或置信度。

## 业务解释
本节回答“这些排名结果在业务上意味着什么”。
建议：
- 解读头部集中度（是否符合帕累托法则？）。
- 分析头部子项的共性特征（品类、价格带、用户群等）。
- 说明尾部表现可能揭示的问题（如长尾效应、市场偏好变化）。

表达：
- 必须基于上文的排名数据和表格进行解释。
- 可以提出假设性业务推断，但需使用“可能表明”、“或许反映了”等措辞。
- 不应将排名结果直接等同于业务成功或失败的原因。

---

## 限制与数据问题
必须单独成节，且内容必须具体。针对分组排名分析，需重点检查并说明：
1.  **分组维度一致性**：主键/分组维度是否存在不同写法、别名或合并问题？这对分组准确性的影响是什么？
2.  **指标口径**：排序指标的计算口径是否清晰？是否包含可能影响排名的特殊记录（如退货、测试单）？
3.  **样本与时间范围**：数据是否覆盖了完整的业务范围？时间范围是否可能影响排名的时效性？
4.  **数据完整性**：是否存在因数据缺失导致的某个分组未参与排名或排名不准确的情况？
5.  **Top N 筛选的局限性**：报告是否仅展示了Top N？如果是，需提示未展示的长尾部分可能包含重要信息。

写法：
- 说明每一条限制具体影响了哪一条排名结论。
- 如果数据问题严重，需明确说明该限制下的排名结果“仅供参考”或“建议修正数据后重新分析”。

---

## 结论与建议

### 结论摘要
- 用2-3句话总结最核心的排名发现。
- 明确当前基于数据的最可信排名是什么。
- 重申关键限制对结论可靠性的影响。

### 后续建议
按优先级排序：
1.  **数据校验建议**：例如，建议对【某个分组维度】进行标准化清洗，以获得更精确的排名。
2.  **深度分析建议**：例如，可对头部子项进行下钻分析，或分析尾部子项的共性。
3.  **业务行动参考**：如果证据足够，可建议针对头部进行资源倾斜。如果证据不足，应明确指出“当前排名结果可作为初步参考，但建议结合【其他信息】后再制定具体行动”。

---

## 写作风格与硬约束

### 写作风格
- 语气中性、直接、证据优先。
- 少写铺垫，多写和用户问题直接相关的发现。
- 少用形容词，多用数字和比较。
- 没有把握时，显式写“当前证据不足”。
- 不要伪精确，不要虚构原因。
- 不要把模板中的所有模块机械写满，应根据用户问题选择重点。
- 核心发现部分的篇幅必须明显多于问题与范围、限制说明等辅助部分。

### 报告生成时的硬约束
1.  **分组一致性前置原则**：在展示任何分组排名结果前，报告必须说明用于分组的关键字段的数据标准化情况或潜在问题。
2.  **单期数据禁用趋势词**：若上游分析仅提供单期（无历史对比）数据，报告中严禁使用“增长”、“下降”、“改善”、“恶化”等趋势性词汇。
3.  **排名结果需附带说明**：任何Top/Bottom排名表格后，必须附带1-2句说明，指出该排名反映的是特定时间段内的数据情况，并可能受数据完整性影响。
4.  **证据强度匹配结论语气**：结论的语气强度（肯定、可能、推测）必须与支撑证据的强度（数据量、一致性、可比性）严格匹配。
5.  **完整数据来源指引**：当核心发现中使用Top N（如前10名）表格时，必须在表格后注明“完整数据集包含全部[N]项记录，详情可参见附件/数据库表[名称]”。
6.  **禁止跨维度绝对比较**：在缺乏可比基准（如价格、规模、市场环境）时，禁止直接进行不同分组间子项排名的绝对值比较。
7.  **禁止强因果表述**：严禁将描述性的排名现象表述为确定的业务因果关系。
8.  **核心发现必须直接回应问题**：不得用通用的经营概览代替针对“排名”问题的直接回答。
9.  **限制说明不可省略**：任何可能影响排名结论可靠性的数据或分析限制，必须明确披露。

## 默认篇幅建议
- 执行摘要：150-250字。
- 问题与范围：80-150字。
- 核心发现：400-700字。
- 核心发现中的局部数据表格：1-3个，每个5-10行。
- 关键证据表：3-6条。
- 业务解释：150-300字。
- 限制与数据问题：80-200字。
- 结论与建议：150-300字。

**调整说明**：
- 对于简单的“找Top N”类问题，可压缩执行摘要和问题范围，将篇幅集中在核心发现、排名表格和解读上。
- 如果用户问题复杂（如要求分析排名变化的原因），则核心发现应包含“排名结果展示”和“潜在原因分析”两个层次。
