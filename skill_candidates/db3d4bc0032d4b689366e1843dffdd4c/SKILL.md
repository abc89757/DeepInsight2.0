# 二手车辆交易分析场景总说明

本 skill 适用于分析二手车（或类似高价值、非标准化二手商品）的交易数据，核心是评估商品的市场接受度、识别畅销商品并理解交易结构。

## 这个场景要回答什么

二手车辆交易分析不只回答“哪款车卖得好”，而是要回答：

- 热度如何：各品牌、车系、具体车型的成交量、成交额表现如何，哪些是市场交易主力。
- 结构特征：销售是集中于少数几个品牌/车型，还是相对分散？不同价格区间、车龄段的销售结构有何差异？
- 潜在关联：畅销车型是否与特定因素（如高性价比、低车龄、特定城市偏好）相关联？
- 比较差异：在不同细分维度下（如不同城市、不同车况等级），畅销车型是否存在显著不同？
- 证据是否充足：当前数据维度（如仅有品牌、车型、成交量）是否足以回答用户问题，还是需要进一步追加车龄、价格等数据才能得出可靠结论。

## 分析职责

你负责先把用户问题归类，再确定本轮主分析方向。

### 常见问题类型与默认方向

- 如果用户问“各品牌卖得最好的车型是哪款” / “哪个车型最畅销”：
  默认先以成交量（销量）为核心排序指标，按品牌分组找出每个品牌的Top 1（或Top N）车型，并列呈现其成交额作为辅助参考。
- 如果用户问“哪个价格段的车最好卖” / “什么车龄的二手车最受欢迎”：
  默认先对车辆按价格区间或车龄进行分组，计算各组的成交量、成交额，再可选地在每个分组内分析热门品牌/车型。
- 如果用户问“某个品牌下哪些车型卖得好”：
  默认聚焦该品牌，分析其旗下所有车型的销售排名和销售结构（集中度）。
- 如果用户问“为什么某些车卖不动” / “畅销车有什么特点”：
  默认需要引入更多维度（如车龄、价格、城市、上牌时间）进行交叉分析，或对比畅销与滞销车型在这些维度上的差异。
- 如果用户问“不同城市的畅销车型一样吗”：
  默认以城市为首要分析维度，分别计算各城市的品牌/车型销量排名，并进行对比。

## 默认分析框架

每次分析优先遵循下面顺序：

1.  先定义分析范围与口径
    - **时间范围**：必须明确数据的时间窗口（如“近一年”、“上个季度”、“2023年全年”）。
    - **业务口径**：必须明确定义“成交”的业务标准（如“已完成付款的订单”、“已过户的车辆”）。
    - **对象范围**：可选定义地域范围（如全国、某个省份/城市）、数据来源范围（如某个平台、多个平台聚合）。

2.  确定核心分析维度
    - **必选维度**：品牌、车型（这是定位具体商品的最小单元）。
    - **常用维度**：车龄段、价格区间、城市、上牌年份。
    - **可选维度**：车况等级、排放标准、变速箱类型、燃油类型（纯电/混动/燃油）、颜色。

3.  选择核心指标与辅助指标
    - **核心排序指标**：**成交量（或销量）**。这是衡量市场热度的最直接指标，用于回答“卖得最好”。
    - **重要辅助指标**：**成交额**。用于衡量销售规模贡献，其排名可能与成交量排名不同，这种差异本身就是洞察。
    - **其他可参考指标**：平均成交价、成交车辆数量（去重后）、成交周期（如果数据支持）。

4.  执行分析与计算
    - 执行“**分组-聚合-排序**”的核心分析路径。例如：按“品牌”分组 -> 计算每组“成交量” -> 按“成交量”降序排列。
    - 对于“找组内Top”的问题，执行“**主维度分组 -> 次维度分组 -> 指标计算 -> 组内排序**”。例如：按“品牌”分组 -> 在品牌内按“车型”分组 -> 计算每个车型的成交量 -> 在每个品牌内对车型按成交量排序。

5.  结果呈现与深度解读
    - **结构化呈现**：采用“总览 -> 分项明细”的结构。先展示整体数据范围和各品牌销量概览（可选），再分品牌列出其最畅销车型的成交量、成交额。
    - **提供解读视角**：结合车龄、价格等维度解读结果。例如，“A品牌最畅销的是XX车型，平均车龄3年，主要在10-15万元价格段成交”。
    - **识别模式与异常**：指出发现的模式（如“豪华品牌畅销车型集中度普遍高于大众品牌”）或异常点（如“B品牌销量第一的车型，其成交额排名却很靠后”）。

## 建议优先级

默认优先回答以下问题：

1.  用户指定维度（如各品牌）下的Top 1（或Top N）车型及其成交量。
2.  这些畅销车型对应的成交额规模。
3.  在现有数据维度下，是否能观察到明显模式（如集中度高/低，或与车龄/价格显著相关）。
4.  当前证据的确定性（数据时间范围、口径是否清晰，维度是否足够解释问题）。
5.  是否需要、以及可能需要哪些额外维度（如车龄、价格）才能得出更稳固的结论。

## 触发继续追证的条件

满足任一条件时，你应优先继续找证据，而不是立刻进入最终报告：

- **发现排名异常**：某品牌的销量冠军车型其成交额排名极低（或反之），可能存在极端低价/高价交易或数据口径问题，需要检查。
- **销量高度集中**：某品牌的销量严重依赖于一两款车型，需要探究这是长期结构还是受短期因素（如一批库存车流入市场）影响。
- **用户问题涉及“为什么”**：当用户问“为什么卖得好”或“特点是什么”时，当前仅有品牌、车型、成交量的证据不足以回答，必须引入车龄、价格、城市等维度进行下钻或对比分析。
- **维度交叉矛盾**：在不同细分维度（如不同城市）下，畅销车型排序差异巨大，无法给出统一结论，需要进一步分析地域偏好。
- **关键信息缺失**：数据缺失时间范围，或“成交”口径未定义，导致分析结果可能产生歧义。

## 必须避免的行为

- **避免唯成交量论**：不能仅凭成交量高就断定“卖得好”且健康，必须结合成交额、平均成交价等指标进行综合判断，识别是否存在“以价换量”的情况。
- **忽略车辆非标特性**：不能将二手车简单等同于标准化商品进行分析。应主动考虑并建议引入车龄、车况、价格区间、历史记录等二手车特有维度，这些维度对交易结果影响巨大。
- **混淆新车与二手车分析逻辑**：避免套用针对新车市场的库存周转、生命周期等分析模型。二手车的库存逻辑、定价逻辑与新车有本质不同。
- **在未明确口径时下结论**：绝不能在未明确分析的时间范围、成交业务口径的情况下，直接给出“哪个品牌/车型卖得最好”的确定性结论。
- **混淆相关性与因果性**：例如，不能因为发现“3年内车龄的车辆成交量最高”，就直接断定“消费者只喜欢买准新车”，这需要排除新车本身流入二手市场数量的影响等因素。

## 输出目标

你的输出应至少包含：

- 本轮分析目标
- 本轮使用的分析维度与核心指标
- 需要的关键数据证据（需要哪些字段）
- 暂不分析的内容与原因（例如，用户未问及价格趋势，故暂不分析）
- 是否需要继续追证，还是可以基于当前证据进入报告生成
