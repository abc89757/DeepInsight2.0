# 分组内最优对象识别场景总说明

本 skill 适用于任何需要将对象按某一维度分组，并在每个组内根据核心指标筛选出最优对象的业务或数据分析场景。

## 场景适用范围

此场景模式广泛存在于：
- **电商零售**：找出各品类、各品牌下销量/销售额最高的商品。
- **客户服务**：识别各服务渠道、各客服小组中客户满意度最高的成员或对话。
- **内容平台**：定位各频道、各作者发布的内容中热度/互动最高的作品。
- **人力资源**：筛选各部门、各职级中绩效评估排名靠前的员工。
- **市场营销**：分析各推广渠道、各活动带来的转化效果最佳的线索或素材。
- **供应链管理**：评估各供应商、各仓库在交付准时率或质量合格率方面的表现。

**边界**：本 Skill 聚焦于基于单一核心指标的简单排序与筛选。当问题涉及多指标权衡、趋势归因、价格策略、库存影响或客户行为深度分析时，应使用更综合的分析 Skill（如“商品销售分析”）。

## 这个场景要回答什么

本场景的核心问题是：**在给定的分组条件下，哪个具体对象在核心衡量指标上表现最优？**

答案直接指向一个或多个明确的“分组 -> 最优对象”的对应关系及其核心指标值。例如：“在‘手机’品类下，销量最高的商品是A；在‘家电’品类下，销量最高的商品是B。”

## 默认分析框架

执行此类分析时，优先遵循以下通用步骤：

1.  **确认分组维度**
    *   识别并明确用户问题中隐含或指定的分组依据。常见维度如：品牌、品类、区域、渠道、时间周期（如季度）、团队等。
    *   **规则**：若分组维度不清晰，必须向用户确认。

2.  **定义“最优”标准**
    *   与用户确认或根据业务上下文明确定义用于排序的“核心指标”。这通常是单一的、可量化的指标。
    *   **常见指标示例**：销量、销售额、成交量、评分、完成率、点击率、利润额、客户满意度（NPS/CSAT）。
    *   **规则**：避免默认使用单一数量指标（如仅销量），需考虑是否应结合其他业务背景（如是否关注利润率）。当用户使用“最好”、“最优”、“最高”等模糊表述时，应优先询问其具体衡量标准，或基于业务常识给出默认建议（如销售场景通常默认为“销量”或“销售额”）并获得确认。

3.  **获取分组-对象-指标数据**
    *   规划并获取包含上述三个关键要素的数据集。数据应能支持分组、计算和排序。
    *   **规则**：若数据不具备分组维度或核心指标字段，或存在大量缺失值、异常值（如负的销量、极高的异常评分），可能影响排名公正性，应考虑追证或修正。

4.  **执行分组内排序与筛选**
    *   对数据按“分组维度”进行分组。
    *   在每个分组内，按“核心指标”进行降序排列（默认行为，可根据业务调整排序方向）。
    *   提取每个分组内排名第一的对象（或用户指定的前N名）。

5.  **呈现结果**
    *   以清晰、直观的方式展示“分组维度”、“最优对象”及其对应的“核心指标值”。
    *   可视化建议：使用表格、条形图（按分组维度对比最优对象的指标值）或分组的列表。

## 继续追证条件

在执行过程中，若出现以下情况，应暂停产出最终结果，优先寻求补充信息或数据验证：
1.  **定义模糊**：用户对“分组维度”或“最优标准”的定义存在多种合理理解，且未明确指定。
2.  **数据缺陷**：用于分组或排序的关键字段缺失率过高，或存在可能扭曲排名的异常值/错误数据。
3.  **标准升级**：用户在查看初步结果后，对“最优”提出了新的或多维度的要求（如“不仅要看销量，还要看好评率”），这需要将新的要求转化为明确的复合指标或排序规则。

## 必须避免的行为

1.  **避免前提不清即行动**：在未明确分组维度和核心排序指标的情况下，不得直接开始数据分析与排名。
2.  **避免指标简化误判**：不得将“最优”默认且唯一地等同于单一的数量指标（如只看销量绝对值），应考虑业务上下文（例如，是否需考虑价格、利润或市场份额）。除非用户明确指定。
3.  **避免具体化绑定**：不得将本次分析任务中的具体分组维度（如汽车品牌）或具体对象（如车型）作为通用规则写入。规则必须是抽象的、可替换的。
4.  **避免过度归因**：对排名结果不得进行超出数据支撑的因果归因（例如，不能直接得出“因为该车型口碑好所以销量高”的结论，这属于后续分析范畴）。本 Skill 的输出应严格限定于描述“是什么”（即排名结果），而非解释“为什么”。

## 输出目标

你的输出应至少包含：
- 本轮分析目标（回答“在...分组下，谁在...指标上最优？”）
- 明确的分组维度与核心排序指标
- 需要获取的关键数据要素（分组字段、对象标识、指标字段）
- 暂不分析的内容与原因（如：不分析指标变动趋势、不分析对象间的关联影响）
- 基于数据完备性和定义清晰度的判断：是直接进入报告生成，还是需要继续追证。
